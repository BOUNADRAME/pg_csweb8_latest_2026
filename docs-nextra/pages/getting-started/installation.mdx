## Installation Rapide (5 minutes)

### Étape 1 : Cloner le Projet

```bash
git clone https://github.com/BOUNADRAME/pg_csweb8_latest_2026.git
cd pg_csweb8_latest_2026
````

### Étape 2 : Configuration Environnement

```bash
# Copier le fichier .env d'exemple
cp .env.example .env

# Éditer .env (IMPORTANT : Changer les mots de passe !)
nano .env  # ou vim, code, etc.
```

**Variables critiques à modifier :**

**Mode Breakout :**
```env
BREAKOUT_MODE=local
BREAKOUT_DB_TYPE=postgresql
```

**Application :**
```env
APP_ENV=dev
APP_DEBUG=true
APP_SECRET=CHANGEZ_CETTE_VALEUR_ALEATOIRE
```

**MySQL - Métadonnées CSWeb (toujours local) :**
```env
MYSQL_HOST=mysql
MYSQL_PORT=3306
MYSQL_DATABASE=csweb_metadata
MYSQL_USER=csweb_user
MYSQL_PASSWORD=CHANGEZ_MOT_DE_PASSE
MYSQL_ROOT_PASSWORD=CHANGEZ_MOT_DE_PASSE_ROOT
```

**PostgreSQL - Base Breakout :**
```env
POSTGRES_HOST=postgres
POSTGRES_PORT=5432
POSTGRES_DATABASE=csweb_analytics
POSTGRES_USER=csweb_analytics
POSTGRES_PASSWORD=CHANGEZ_MOT_DE_PASSE
```

**Générer des secrets aléatoires :**

```bash
# Linux/macOS
openssl rand -hex 32
```

### Étape 3 : Démarrer les Services Docker

```bash
# Démarrer avec le profil PostgreSQL local (développement)
docker compose --profile local-postgres up -d --build

# Vérifier que tous les conteneurs tournent
docker compose ps
```

**Sortie attendue :**

```
NAME                      STATUS              PORTS
csweb-app                 Up (healthy)        0.0.0.0:8080->80/tcp
csweb-mysql-metadata      Up (healthy)        0.0.0.0:3306->3306/tcp
csweb-postgres-breakout   Up (healthy)        0.0.0.0:5432->5432/tcp
```

> **Note :** Le conteneur `csweb-app` peut afficher `(unhealthy)` avant la configuration initiale.
> C'est normal car le healthcheck teste l'API qui n'est pas encore configurée.

### Étape 4 : Accéder à CSWeb

Ouvrez votre navigateur et accédez à :

**CSWeb Application :**

```
http://localhost:8080
```

Vous serez automatiquement redirigé vers `/setup/` pour la configuration initiale.

## Configuration Initiale via Setup

### Étape 5 : Wizard de Configuration

Après le premier démarrage, CSWeb vous redirige vers le wizard de configuration :

1. **Page Requirements** (`/setup/`) :
   - Vérifie PHP 8.1+, extensions PDO, MySQLi, cURL, etc.
   - Tous les checks doivent être verts
   - Cliquer sur **"Next"**

2. **Page Configuration** (`/setup/configure.php`) :

   | Champ                       | Valeur                  | Description                                          |
   | --------------------------- | ----------------------- | ---------------------------------------------------- |
   | **Database name**           | `csweb_metadata`        | Nom de la base MySQL métadonnées                     |
   | **Hostname**                | `mysql`                 | Nom du conteneur MySQL (pas localhost !)             |
   | **Database username**       | `root`                  | Utilisateur MySQL (root nécessaire pour créer la DB) |
   | **Database password**       | `[MYSQL_ROOT_PASSWORD]` | Le mot de passe root défini dans `.env`              |
   | **CSWeb admin password**    | `[MOT_DE_PASSE_FORT]`   | Minimum 8 caractères                                 |
   | **Timezone**                | `UTC`                   | Ou votre fuseau horaire                              |
   | **Maximum execution time**  | `300`                   | En secondes                                          |
   | **Path to files directory** | `/var/www/html/files`   | Chemin par défaut                                    |
   | **CSWeb API URL**           | `http://localhost/api/` | **Port 80 interne, PAS 8080 !**                      |

   > **Important :** L'API URL doit être `http://localhost/api/` (sans le port 8080).
   > À l'intérieur du conteneur Docker, Apache écoute sur le port **80**.
   > Le port 8080 est uniquement le mapping externe visible dans votre navigateur.

3. **Cliquer sur "Next"** pour finaliser l'installation.

4. **Connexion :**
   - Username : `admin`
   - Password : le mot de passe admin choisi à l'étape précédente

## Setup Mode Remote (Serveur Distant)

Le mode **remote** permet de connecter CSWeb à un serveur de base de données **distant** (PostgreSQL, MySQL ou SQL Server), sans conteneur breakout local.

### Quand utiliser le mode Remote ?

- **Production** : Séparation metadata/analytics sur des serveurs différents
- **RGPH / Recensement** : Base analytics sur un serveur dédié haute performance
- **Équipe distribuée** : Plusieurs instances CSWeb pointant vers la même base analytics

### Fonctionnement

En mode remote :

1. Le **pilote par défaut** (`BREAKOUT_DB_TYPE`) dans `.env` détermine le driver PDO utilisé (PostgreSQL, MySQL ou SQL Server)
2. La **connexion à la base distante** (host, user, password, database) est configurée **au moment du breakout** dans l'interface CSWeb
3. CSWeb supporte les 3 types de bases : vous choisissez le pilote selon votre serveur distant

### Étape 1 : Configurer le `.env`

```env
# Passer en mode remote
BREAKOUT_MODE=remote

# Pilote par défaut - détermine le driver PDO utilisé
# postgresql | mysql | sqlserver
BREAKOUT_DB_TYPE=postgresql

# MySQL Metadata reste LOCAL (container Docker)
MYSQL_HOST=mysql
MYSQL_PORT=3306
MYSQL_DATABASE=csweb_metadata
```

> **Note :** Les informations de connexion au serveur distant (host, user, password)
> seront renseignées dans CSWeb lors de la configuration du breakout par dictionnaire.

### Étape 2 : Démarrer sans conteneur breakout

```bash
# En mode remote, ne PAS utiliser le profil local-postgres
# Démarrer uniquement CSWeb + MySQL
docker compose up -d csweb mysql

# Vérifier les conteneurs
docker compose ps
```

**Sortie attendue :**

```
NAME                      STATUS              PORTS
csweb-app                 Up (healthy)        0.0.0.0:8080->80/tcp
csweb-mysql-metadata      Up (healthy)        0.0.0.0:3306->3306/tcp
```

> Pas de conteneur PostgreSQL/SQL Server - la connexion se fait vers le serveur distant.

### Étape 3 : Vérifier les drivers disponibles

```bash
# Vérifier que le driver correspondant au pilote choisi est installé
docker compose exec csweb php bin/console csweb:check-database-drivers
```

**Sortie attendue :**

```
 POSTGRESQL   ✅ Available
 MYSQL        ✅ Available
 SQLSERVER    ✅ Available

 [OK] All database types are supported (3/3)
```

### Étape 4 : Configurer le breakout dans CSWeb

La connexion au serveur distant se configure **dans l'interface CSWeb** :

1. Se connecter à `http://localhost:8080`
2. Aller dans **Settings > Data**, onglet **Data Configuration**
3. Cliquer **Add Configuration** et sélectionner un dictionnaire
4. Renseigner les informations du serveur distant :
   - **Database name** : Nom de la base breakout (doit être différente de la base métadonnées)
   - **Hostname** : IP ou hostname du serveur distant
   - **Port** : Port du SGBD (5432, 3306, 1433)
   - **Username / Password** : Credentials du serveur distant

Chaque dictionnaire peut pointer vers un serveur et un driver différent.

### Prérequis Serveur Distant

Le serveur distant doit être accessible depuis le conteneur CSWeb :

**PostgreSQL :**

```ini
# postgresql.conf
listen_addresses = '*'
```

```
# pg_hba.conf
host    all    all    0.0.0.0/0    md5
```

**MySQL :**

```ini
# my.cnf
bind-address = 0.0.0.0
```

**SQL Server :**

- TCP/IP activé dans SQL Server Configuration Manager
- Port 1433 ouvert dans le firewall

**Firewall :**

```bash
# PostgreSQL
sudo ufw allow from [IP_SERVEUR_CSWEB] to any port 5432

# MySQL
sudo ufw allow from [IP_SERVEUR_CSWEB] to any port 3306

# SQL Server
sudo ufw allow from [IP_SERVEUR_CSWEB] to any port 1433
```

### Basculer entre Local et Remote

```bash
# Passer en local
# 1. Modifier .env : BREAKOUT_MODE=local
# 2. Relancer avec le profil
docker compose --profile local-postgres up -d

# Passer en remote
# 1. Modifier .env : BREAKOUT_MODE=remote
# 2. Relancer sans profil breakout
docker compose up -d csweb mysql
```

## Architecture Docker

### Services Déployés

```
┌─────────────────────────────────────────────────┐
│              Docker Compose Stack                │
│                                                  │
│  ┌──────────┐  ┌──────────┐  ┌──────────────┐  │
│  │  CSWeb   │  │  MySQL   │  │  PostgreSQL  │  │
│  │ (Apache) │  │(Metadata)│  │ (Analytics)  │  │
│  │  :8080   │  │  :3306   │  │    :5432     │  │
│  └──────────┘  └──────────┘  └──────────────┘  │
│                                                  │
│            csweb-network (bridge)                │
│                                                  │
│  ┌──────────┐  ┌──────────┐                     │
│  │phpMyAdmin│  │ pgAdmin  │  (profil dev)       │
│  │  :8081   │  │  :8082   │                     │
│  └──────────┘  └──────────┘                     │
└─────────────────────────────────────────────────┘
```

### Profils Docker Disponibles

| Profil              | Commande                                         | Services                       |
| ------------------- | ------------------------------------------------ | ------------------------------ |
| **local-postgres**  | `docker compose --profile local-postgres up -d`  | CSWeb + MySQL + PostgreSQL     |
| **local-mysql**     | `docker compose --profile local-mysql up -d`     | CSWeb + MySQL + MySQL Breakout |
| **local-sqlserver** | `docker compose --profile local-sqlserver up -d` | CSWeb + MySQL + SQL Server     |
| **dev**             | `docker compose --profile dev up -d`             | + phpMyAdmin + pgAdmin         |

### Volumes Docker

Les données sont persistées dans des volumes Docker :

```bash
# Lister les volumes
docker volume ls | grep csweb

# Backup PostgreSQL
docker compose exec postgres pg_dump -U csweb_analytics csweb_analytics > backup.sql
```

## Commandes Docker Utiles

### Gestion des Services

```bash
# Démarrer les services (profil PostgreSQL local)
docker compose --profile local-postgres up -d

# Arrêter tous les services
docker compose down

# Redémarrer un service spécifique
docker compose restart csweb

# Voir les logs d'un service
docker compose logs -f csweb

# Rebuild après modification du code
docker compose --profile local-postgres up -d --build
```

### Maintenance

```bash
# Entrer dans le conteneur CSWeb
docker compose exec csweb bash

# Vider le cache Symfony (automatique au démarrage via docker-entrypoint.sh)
docker compose exec csweb php bin/console cache:clear

# Vérifier l'utilisation des ressources
docker stats
```

> **Note :** Le `docker-entrypoint.sh` gère automatiquement au démarrage :
> le vidage du cache Symfony, l'installation du cron scheduler (breakout automatique),
> et la suppression de la contrainte `schema_name` si nécessaire.

## Troubleshooting

### Port 8080 déjà utilisé

```bash
# Trouver le processus utilisant le port
sudo lsof -i :8080

# Tuer le processus
sudo kill -9 [PID]

# OU changer le port dans .env
CSWEB_PORT=8888
```

### HTTP 500 après configuration

Le cache Symfony est automatiquement vidé au démarrage du conteneur via `docker-entrypoint.sh`. Si le problème persiste :

```bash
docker compose exec csweb php bin/console cache:clear
docker compose exec csweb chmod -R 777 /var/www/html/var
```

### Access denied lors du setup

Le setup CSWeb nécessite un utilisateur MySQL avec les privilèges `CREATE DATABASE`.
Utilisez `root` comme Database username dans le wizard de configuration.

### API URL - "Failed to connect to localhost port 8080"

L'API URL dans le wizard doit être `http://localhost/api/` (port 80 interne au conteneur).
Le port 8080 n'est accessible que depuis votre navigateur (mapping externe).

### Container ne démarre pas (unhealthy)

```bash
# Vérifier les logs
docker compose logs csweb
docker compose logs postgres

# Vérifier la santé
docker inspect csweb-app | grep -A 20 Health
```
