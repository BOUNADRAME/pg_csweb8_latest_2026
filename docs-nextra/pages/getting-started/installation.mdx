# Installation CSWeb Community Platform

Ce guide vous permettra d'installer et de démarrer CSWeb Community Platform en **5 minutes** avec Docker.

<Callout type="info">
  **Temps estimé :** 5-10 minutes (téléchargement Docker images inclus)
</Callout>

<Callout>
  **Base officielle :** CSWeb Community Platform est basé sur **CSWeb 8** téléchargé depuis le site officiel [csprousers.org/downloads](https://csprousers.org/downloads/)
</Callout>

---

## Installation Rapide (5 minutes)

### Étape 1 : Cloner le Projet

```bash
git clone https://github.com/BOUNADRAME/pg_csweb8_latest_2026.git
cd pg_csweb8_latest_2026
```

### Étape 2 : Configuration Environnement

```bash
# Copier le fichier .env d'exemple
cp .env.example .env

# Éditer .env (IMPORTANT : Changer les secrets !)
nano .env  # ou vim, code, etc.
```

**Variables critiques à modifier :**

```env
# SÉCURITÉ - GÉNÉRER DES VALEURS ALÉATOIRES
APP_SECRET=CHANGEZ_CETTE_VALEUR_32_CARACTERES
DATABASE_URL="mysql://csweb_user:CHANGEZ_MOT_DE_PASSE@csweb_mysql:3306/csweb_db"

# POSTGRESQL (Base Breakout)
POSTGRES_DB=csweb_analytics
POSTGRES_USER=csweb_user
POSTGRES_PASSWORD=CHANGEZ_MOT_DE_PASSE
POSTGRES_HOST=csweb_postgres
POSTGRES_PORT=5432

# MYSQL (Base Metadata CSWeb)
MYSQL_ROOT_PASSWORD=CHANGEZ_MOT_DE_PASSE_ROOT
MYSQL_DATABASE=csweb_db
MYSQL_USER=csweb_user
MYSQL_PASSWORD=CHANGEZ_MOT_DE_PASSE
```

<Callout type="error">
  **IMPORTANT :** Ne laissez JAMAIS les mots de passe par défaut en production ! Générez des valeurs aléatoires de 32 caractères minimum.
</Callout>

**Générer des secrets aléatoires :**

```bash
# Linux/macOS
openssl rand -hex 32

# OU
head -c 32 /dev/urandom | base64
```

### Étape 3 : Démarrer les Services Docker

```bash
# Démarrer tous les services en arrière-plan
docker-compose up -d

# Vérifier que tous les containers tournent
docker-compose ps
```

**Sortie attendue :**

```
NAME                  STATUS              PORTS
csweb_app            Up (healthy)        0.0.0.0:8080->80/tcp
csweb_mysql          Up (healthy)        0.0.0.0:3306->3306/tcp
csweb_postgres       Up (healthy)        0.0.0.0:5432->5432/tcp
phpmyadmin           Up                  0.0.0.0:8081->80/tcp
pgadmin              Up                  0.0.0.0:8082->80/tcp
```

<Callout type="success">
  Si tous les containers sont "Up (healthy)", l'installation a réussi ! ✅
</Callout>

### Étape 4 : Accéder à CSWeb

Ouvrez votre navigateur et accédez à :

**CSWeb Application :**
```
http://localhost:8080
```

**Outils Admin (Développement) :**
- phpMyAdmin (MySQL) : http://localhost:8081
- pgAdmin (PostgreSQL) : http://localhost:8082

<Callout type="info">
  **Connexion par défaut :**
  - Username : `admin`
  - Password : Défini dans `.env` (variable `CSWEB_ADMIN_PASSWORD`)
</Callout>

## Configuration Initiale via Setup.php

### Étape 5 : Wizard de Configuration

Après le premier démarrage, vous devez configurer CSWeb via `setup.php` :

1. **Accéder au wizard :**
   ```
   http://localhost:8080/setup/
   ```

2. **Vérification des prérequis :**
   - PHP 8.1+ ✅
   - Extensions requises (PDO, MySQLi, cURL) ✅
   - Permissions d'écriture ✅

3. **Configuration Base de Données :**
   ```
   Database Host: csweb_mysql
   Database Name: csweb_db
   Username: csweb_user
   Password: [MOT_DE_PASSE_MYSQL_ENV]
   ```

4. **Compte Admin :**
   - Username : `admin`
   - Email : votre@email.com
   - Password : [CHOISIR_MOT_DE_PASSE_FORT]

5. **Finaliser :**
   - Cliquer sur "Install"
   - Patienter 30-60 secondes
   - Redirection automatique vers login

<Callout type="warning">
  **Sécurité :** Après l'installation, supprimez ou renommez le dossier `setup/` pour éviter tout accès non autorisé.
</Callout>

```bash
# Lancer la commande de vérification des drivers

```

```
✅ MySQL (pdo_mysql): Available
```

## Architecture Docker

### Services Déployés

```
┌────────────────────────────────────────────────────┐
│              Docker Compose Stack                   │
├────────────────────────────────────────────────────┤
│                                                     │
│  ┌─────────────┐   ┌──────────┐   ┌────────────┐  │
│  │   CSWeb     │───│  MySQL   │   │ PostgreSQL │  │
│  │  (Apache)   │   │(Metadata)│   │ (Analytics)│  │
│  │   :8080     │   │  :3306   │   │   :5432    │  │
│  └─────────────┘   └──────────┘   └────────────┘  │
│         │                                           │
│  ┌──────────────────────────────────────────────┐  │
│  │        csweb-network (bridge)                │  │
│  └──────────────────────────────────────────────┘  │
│                                                     │
│  ┌───────────┐              ┌────────────┐         │
│  │phpMyAdmin │              │  pgAdmin   │         │
│  │   :8081   │              │   :8082    │         │
│  └───────────┘              └────────────┘         │
│  (dev only)                 (dev only)             │
└────────────────────────────────────────────────────┘

Volumes Persistants:
├── csweb_files      (fichiers uploadés)
├── csweb_logs       (logs application)
├── mysql_data       (données MySQL)
└── postgres_data    (données PostgreSQL)
```

### Volumes Docker

Les données sont persistées dans des volumes Docker :

```bash
# Lister les volumes
docker volume ls | grep csweb

# Inspecter un volume
docker volume inspect pg_csweb8_latest_2026_postgres_data

# Backup d'un volume
docker run --rm -v pg_csweb8_latest_2026_postgres_data:/data -v $(pwd):/backup ubuntu tar czf /backup/postgres_backup.tar.gz /data
```

## Commandes Docker Utiles

### Gestion des Services

```bash
# Démarrer tous les services
docker-compose up -d

# Arrêter tous les services
docker-compose down

# Redémarrer un service spécifique
docker-compose restart csweb_app

# Voir les logs d'un service
docker-compose logs -f csweb_app

# Voir les logs de tous les services
docker-compose logs -f

# Rebuild après modification
docker-compose up -d --build
```

### Maintenance

```bash
# Entrer dans le container CSWeb
docker exec -it csweb_app bash

# Entrer dans PostgreSQL
docker exec -it csweb_postgres psql -U csweb_user -d csweb_analytics

# Entrer dans MySQL
docker exec -it csweb_mysql mysql -u root -p

# Vérifier l'utilisation des ressources
docker stats

# Nettoyer les containers arrêtés
docker container prune

# Nettoyer les volumes non utilisés
docker volume prune
```

## Modes de Déploiement

### Mode Développement (Par défaut)

Inclut phpMyAdmin et pgAdmin pour faciliter le développement :

```bash
docker-compose up -d
```

### Mode Production

Exclut les outils de développement pour la sécurité :

```bash
docker-compose --profile production up -d
```

- ❌ Pas de phpMyAdmin

## Troubleshooting

### Port 8080 déjà utilisé

**Erreur :**
```
Error starting userland proxy: listen tcp4 0.0.0.0:8080: bind: address already in use
```

**Solution :**
```bash
# Trouver le processus utilisant le port
sudo lsof -i :8080

# Tuer le processus
sudo kill -9 [PID]

# OU changer le port dans docker-compose.yml
ports:
  - "8888:80"  # Au lieu de 8080:80
```

### Container ne démarre pas (unhealthy)

**Vérifier les logs :**
```bash
docker-compose logs csweb_app
docker-compose logs csweb_postgres
```

**Vérifier la santé :**
```bash
docker inspect csweb_app | grep -A 20 Health
```

### Permission denied sur volumes

**Linux/macOS :**
```bash
# Corriger les permissions
sudo chown -R $USER:$USER .
chmod -R 755 storage/
```

### Database connection refused

**Vérifier que les containers sont sur le même réseau :**
```bash
docker network inspect pg_csweb8_latest_2026_csweb-network
```

**Vérifier les variables d'environnement :**
```bash
docker exec csweb_app env | grep DATABASE
```
