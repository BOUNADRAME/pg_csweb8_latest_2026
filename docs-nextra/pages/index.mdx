## Qu'est-ce que CSPro et CSWeb ?

**CSPro** (Census and Survey Processing System) est le logiciel officiel du **U.S. Census Bureau** pour la collecte de données de recensements et d'enquêtes statistiques. Utilisé par plus de 100 instituts statistiques dans le monde.

**CSWeb** est le serveur web officiel qui accompagne CSPro pour recevoir, stocker et synchroniser les questionnaires collectés sur tablettes Android.

> **Note :** CSWeb vanilla (version officielle) nécessite une installation manuelle multi-composants (Apache, MySQL, PHP) et une expertise technique pour la configuration. CSWeb Community Platform automatise ce processus avec Docker.

---

## Pourquoi CSWeb Community Platform ?

**Le défi :** Déployer CSWeb vanilla nécessite plusieurs jours de configuration manuelle (Apache, MySQL, PHP, cron) et une expertise technique approfondie.

**Notre solution :** CSWeb Community Platform simplifie ce processus avec **Docker (recommandé)** ou installation manuelle classique, tout en ajoutant des fonctionnalités enterprise-grade : multi-DB, breakout sélectif, scheduler automatique.

## Deux Options d'Installation

### 🐳 Docker (Recommandé)

Setup automatisé en **5 minutes** avec tous les services pré-configurés.

```bash
git clone https://github.com/BOUNADRAME/csweb-community.git
cd csweb-community
docker compose --profile local-postgres up -d
```

## Innovations Principales

### 🎯 Breakout Sélectif par Dictionnaire

**Le problème :** Le breakout global traite TOUS les dictionnaires en même temps, ce qui est lent et risqué pour les gros volumes.

**Notre solution :** Breakez UN dictionnaire à la fois, en parallèle si nécessaire.

```bash
php bin/console csweb:process-cases-by-dict EVAL_PRODUCTEURS
```

**Impact réel - RGPH5 Sénégal :** Traitement de l'ensemble des questionnaires du recensement en une fraction du temps grâce au breakout ciblé.

[📖 Guide Complet](/guides/breakout/selective)

### 🗄️ Architecture Flexible Multi-DB

**Le problème :** CSWeb vanilla est limité à MySQL uniquement.

**Notre solution :** Breakout vers PostgreSQL, MySQL ou SQL Server.

- **Par dictionnaire** : Chaque dictionnaire → sa propre base breakout
- **Local/Remote** : Base sur le même serveur ou serveur distant dédié
- **Cohabitation** : Plusieurs bases simultanées (PostgreSQL + MySQL + SQL Server)

PostgreSQL pour performance analytics, MySQL pour compatibilité, SQL Server pour écosystème Microsoft.

[📖 Architecture Détaillée](/guides/architecture)

### 📡 Webhooks & Monitoring

**Le problème :** Pas de visibilité en temps réel sur la collecte et le traitement des données.

**Notre solution :** Webhooks intégrés + monitoring temps réel pour automatiser vos workflows.

- Notifications sur événements CSPro (upload, sync, breakout)
- Logs centralisés accessibles via interface web
- Métriques et alertes configurables

[📖 Guide Webhooks](/guides/admin/webhooks)
