## Qu'est-ce que le Breakout Sélectif ?

### Définition

Le **breakout sélectif** permet de lancer le processus de breakout pour **un seul dictionnaire CSPro** au lieu de traiter tous les dictionnaires de la base de données CSWeb.

### Comparaison Avant/Après

## Problème Résolu

### Limites CSWeb Vanilla

**Scénario Problématique :**

Vous gérez un projet avec **5 dictionnaires CSPro** différents :
1. `ENQUETE_MENAGES` (150k questionnaires)
2. `RECENSEMENT_POPULATION` (3M questionnaires)
3. `EVALUATION_AGRICOLE` (80k questionnaires)
4. `SONDAGE_SANTE` (50k questionnaires)
5. `TEST_PILOTE` (500 questionnaires)

**Avec CSWeb Vanilla :**

```bash
# Commande unique pour tout breaker
php bin/console csweb:process-all

# Résultat:
# ❌ 48 heures de traitement (3M + 150k + 80k + 50k + 500)
# ❌ Si RECENSEMENT échoue → TOUT échoue
# ❌ Impossible de breaker seulement TEST_PILOTE
# ❌ Tables mélangées dans un seul schéma
```

**Avec CSWeb Community (Breakout Sélectif) :**

```bash
# Breaker seulement TEST_PILOTE (pour test rapide)
php bin/console csweb:process-cases-by-dict TEST_PILOTE
# ✅ 30 secondes

# Breaker RECENSEMENT seul (prod)
php bin/console csweb:process-cases-by-dict RECENSEMENT_POPULATION
# ✅ 6 heures (au lieu de 48h)
# ✅ Si échoue, les autres dictionnaires sont intacts
# ✅ Tables isolées: recensement_cases, recensement_level_1, etc.
```

## Architecture

### Nomenclature des Tables

**Pattern :**
```
{label_dictionnaire}_{nom_table}
```

**Exemple avec dictionnaire `EVAL_PRODUCTEURS_USAID` :**

```
Label du dictionnaire: eval_producteurs
(minuscules, espaces → underscore, suffixe _DICT supprimé)

Tables générées:
├── eval_producteurs_cases          # Cases principales
├── eval_producteurs_notes          # Notes terrain
├── eval_producteurs_cspro_jobs     # Jobs de traitement
├── eval_producteurs_cspro_meta     # Métadonnées
├── eval_producteurs_producteurs    # Level 1 (ménages)
├── eval_producteurs_observations   # Level 2 (parcelles)
└── eval_producteurs_photos         # Level 3 (images)
```

## Installation & Configuration

### Prérequis

✅ CSWeb Community Platform v2.0 installé
✅ PostgreSQL 12+ ou MySQL 8+ configuré
✅ Au moins 1 dictionnaire CSPro uploadé

### Configuration .env

```bash
# Mode breakout
BREAKOUT_MODE=local           # ou remote
BREAKOUT_DB_TYPE=postgresql   # ou mysql, sqlserver

# PostgreSQL (Recommandé pour breakout sélectif)
POSTGRES_HOST=localhost
POSTGRES_PORT=5432
POSTGRES_DATABASE=csweb_analytics
POSTGRES_USER=csweb_user
POSTGRES_PASSWORD=SecurePassword123!
```

### Vérification Installation

```bash
# 1. Vérifier drivers disponibles
docker-compose exec csweb php bin/console csweb:check-database-drivers

# Sortie attendue:
# ✅ pdo_mysql: Available
# ✅ pdo_pgsql: Available
# ✅ pdo_sqlsrv: Available (si SQL Server configuré)

# 2. Lister dictionnaires disponibles
docker-compose exec csweb php bin/console csweb:list-dictionaries

# Sortie attendue:
# ID  | Label                    | Name
# ----|--------------------------|---------------------------------
# 1   | eval_producteurs         | EVAL_PRODUCTEURS_USAID
# 2   | rgph5_menages            | RGPH5_MENAGES_V2
# 3   | test_enquete             | TEST_ENQUETE_PILOTE
```

## Utilisation

### Commande CLI Principale

```bash
php bin/console csweb:process-cases-by-dict <DICTIONARY_NAME> [OPTIONS]
```

### Arguments & Options

| Argument/Option | Type | Description | Exemple |
|----------------|------|-------------|---------|
| `DICTIONARY_NAME` | Argument | Nom exact du dictionnaire | `EVAL_PRODUCTEURS_USAID` |
| `--dict-id=ID` | Option | Alternative : ID du dictionnaire | `--dict-id=3` |
| `--limit=N` | Option | Limiter à N cases (test) | `--limit=100` |
| `--verbose` | Option | Logs détaillés | `-vvv` |
| `--no-interaction` | Option | Mode non-interactif (cron) | `-n` |

### Exemples d'Utilisation

#### Exemple 1 : Breakout Simple

```bash
# Breaker dictionnaire EVAL_PRODUCTEURS_USAID
docker-compose exec csweb php bin/console csweb:process-cases-by-dict EVAL_PRODUCTEURS_USAID
```

### Via API REST

<Callout type="info">
  **Nouveau dans v2.0 :** Breakout sélectif disponible via API REST (OAuth2 requis).
</Callout>

#### Endpoint

```
POST /api/breakout/{dictionary}/trigger
```

#### Exemple cURL

```bash
# 1. Obtenir token OAuth2
curl -X POST http://localhost:8080/api/token \
  -d "grant_type=password" \
  -d "username=admin" \
  -d "password=admin123" \
  -d "client_id=csweb_client"

# Réponse:
# { "access_token": "eyJ0eXAi...", "expires_in": 3600 }

# 2. Lancer breakout
curl -X POST http://localhost:8080/api/breakout/EVAL_PRODUCTEURS_USAID/trigger \
  -H "Authorization: Bearer eyJ0eXAi..." \
  -H "Content-Type: application/json" \
  -d '{ "limit": null, "verbose": true }'

# Réponse:
# {
#   "status": "success",
#   "job_id": "breakout_job_12345",
#   "dictionary": "EVAL_PRODUCTEURS_USAID",
#   "started_at": "2026-03-15T10:30:00Z"
# }

# 3. Vérifier statut
curl http://localhost:8080/api/breakout/jobs/breakout_job_12345 \
  -H "Authorization: Bearer eyJ0eXAi..."

# Réponse:
# {
#   "job_id": "breakout_job_12345",
#   "status": "completed",
```

## Performance

### Benchmarks

**Configuration Test :**
- **Hardware** : 8 cores CPU, 16GB RAM, SSD
- **Database** : PostgreSQL 16
- **Dictionnaire** : EVAL_PRODUCTEURS (125 cases, 3 levels)

| Opération | Breakout Global | Breakout Sélectif | Gain |
|-----------|----------------|-------------------|------|
| **Cleanup** | 15.2s (toutes tables) | 2.3s (tables dict uniquement) | **6.6x** |
| **Extraction** | 45.1s (tous dicts) | 8.7s (1 dict) | **5.2x** |
| **Insertion** | 120.5s (tous dicts) | 18.7s (1 dict) | **6.4x** |
| **Total** | **180.8s** (3 min) | **29.7s** (30 sec) | **6.1x** |

### Optimisations

#### 1. Index PostgreSQL

```sql
-- Créer indexes après breakout (améliore queries 10x)
CREATE INDEX idx_cases_key ON eval_producteurs_cases(key);
CREATE INDEX idx_cases_verified ON eval_producteurs_cases(verified);
CREATE INDEX idx_producteurs_region ON eval_producteurs_producteurs(region);
```

#### 2. Parallel Processing (Futur v2.1)

```bash
# Breaker 3 dictionnaires en parallèle (3 terminaux)
# Terminal 1
php bin/console csweb:process-cases-by-dict DICT_1 &

# Terminal 2
php bin/console csweb:process-cases-by-dict DICT_2 &

# Terminal 3
php bin/console csweb:process-cases-by-dict DICT_3 &

# Temps total = max(DICT_1, DICT_2, DICT_3) au lieu de sum
```

## Troubleshooting

### Erreur : Dictionary Not Found

**Symptôme :**

```
[ERROR] Dictionary "EVAL_PROD" not found
```

**Cause :** Nom de dictionnaire incorrect.

**Solution :**

```bash
# Lister tous les dictionnaires disponibles
docker-compose exec csweb php bin/console csweb:list-dictionaries

# Utiliser le nom EXACT affiché
php bin/console csweb:process-cases-by-dict EVAL_PRODUCTEURS_USAID
```

---

### Erreur : Database Connection Failed

**Symptôme :**

```
SQLSTATE[HY000] [2002] Connection refused
```

**Cause :** PostgreSQL/MySQL non accessible.

**Solution :**

```bash
# Vérifier services Docker
docker-compose ps

# Vérifier connexion DB
docker-compose exec csweb php bin/console csweb:check-database-drivers --test-connections

# Si PostgreSQL down
docker-compose restart postgres
```

---

### Erreur : Tables Already Exist

**Symptôme :**

```
[ERROR] Table 'eval_producteurs_cases' already exists
```

**Cause :** Breakout précédent non nettoyé.

**Solution :**

```bash
# Option 1: Supprimer manuellement tables
psql -d csweb_analytics
DROP TABLE eval_producteurs_cases CASCADE;
DROP TABLE eval_producteurs_producteurs CASCADE;
-- ... (toutes les tables du dictionnaire)

# Option 2: Re-run breakout (nettoyage auto)
php bin/console csweb:process-cases-by-dict EVAL_PRODUCTEURS_USAID
# Le cleanup est automatique
```

---

### Performance Lente

**Symptôme :**
Breakout très lent (> 10 min pour 1000 cases).

**Solutions :**

```bash
# 1. Vérifier RAM disponible
free -h

# 2. Optimiser PostgreSQL
vim /etc/postgresql/16/main/postgresql.conf

# Augmenter buffers
shared_buffers = 4GB
work_mem = 50MB
maintenance_work_mem = 1GB

# Redémarrer
sudo systemctl restart postgresql

# 3. Créer indexes APRÈS breakout (pas pendant)
# Indexes ralentissent l'insertion mais accélèrent queries
```
